## Технические требования (единые для всех анимаций)

- **Формат:** Lottie JSON (экспорт из After Effects → Bodymovin, или LottieFiles)
- **Альтернатива:** Rive (.riv) — если нужны состояния idle/active
- **Холст:** 400×400 px
- **Длительность:** 2–4 секунды, **бесшовный loop**
- **Фон:** прозрачный (rgba 0,0,0,0)
- **Именование файла:** совпадает с ID упражнения (см. таблицы)

**Цвета Горо:**

| Элемент | HEX |
|---------|-----|
| Тело (тёмное) | `#38384C` |
| Тело (блики) | `#424258` |
| Лицо / кожа | `#C4A882` |
| Повязка | `#A8D8FF` |

**Что показывать:**
- Полный цикл движения: старт → финиш → возврат
- Акцент на ключевом техническом моменте (указан в таблицах)
- Минимум фона: пол (тонкая линия), при необходимости стена/перекладина
- Не показывать «как делать неправильно»

Если Горо в конкретном упражнении выглядит неестественно — согласуй замену на нейтральный силуэт (тёмно-серый, той же толщины). Допустимо предложить альтернативный угол если он лучше читается.

**Для Flex-ветки (статика/растяжка):**
- Движение медленнее, чем в силовых ветках
- Акцент на положении тела, а не скорости
- Допустимо замедленное покачивание / дыхательное движение в удерживаемых позах

Анимации используются в **двух местах**: экран тренировки (основной) и экран прогрессии ветки (превью-миниатюры). Файлы одни и те же.

## Вопросы / решения по ходу работы

Если возникают вопросы по конкретному упражнению (как изобразить технику, с какого угла,
какую позу Горо использовать) — лучше уточнить до начала анимации, чтобы не переделывать.

Допустимо предложить альтернативный угол обзора если он лучше читается.
Допустимо упростить позу Горо если исходная анатомия не ложится на упражнение.
Для блока H (supplementary) допустимо использовать нейтральный силуэт вместо Горо.
