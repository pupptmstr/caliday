## Project Description

A mobile app for home workouts with game-like progression mechanics.
The user progresses from basic exercises (knee push-ups) to advanced
calisthenics (handstand push-ups) through short daily sets of 5вЂ“15 minutes.

**Principles:** all data is local, no backend, free, no ads.

## Key Concepts

### Set (Set / Workout)
A short session: 3вЂ“6 exercises, 5вЂ“15 minutes. Types: `Daily`, `Skill`, `Challenge`.

### Courses
CaliDay supports multiple **courses** (like Duolingo). Each course has its own branch set and progression:
- `CourseId.calisthenics` в†’ push, pull, core, legs, balance, flex
- `CourseId.healthyBody` в†’ posture, neck, flex

**Progression is global per branch:** `SkillProgress` keys are `branch.name` only (e.g. `"push"`). Branches are physical skills вЂ” progress is shared across all courses containing that branch.
**Streak and SP are global.** Active course is stored in `UserProfile.activeCourse` (last selected).
Switching courses happens in the Library tab via pill tabs.

### Progression Branches
8 branches total across all courses. Pull requires a pull-up bar (`requiresEquipment = true`).
`UserProfile.activeBranches` deduplicates branches across all enrolled courses.

### In-Stage Progression
Reps в†‘ в†’ Sets в†‘ (with reps reset) в†’ Rest в†“ в†’ Challenge test в†’ Next stage.

### Gamification
- **SP (Strength Points)** вЂ” points earned for exercises
- **Streak** вЂ” consecutive days; freezes (max 3, earned every 7 streak days)
- **Ranks:** Beginner в†’ Amateur в†’ Athlete в†’ Champion в†’ Master в†’ Legend
- **Achievements** вЂ” 27 total, checked after each workout and stage advance
- **Bonus workouts** вЂ” multiple workouts per day are allowed (50% SP, progression does not advance)

### Primary vs Bonus Workout
`WorkoutLog.isPrimary`: true = first of the day (full SP, advances progression, counts toward streak); false = 50% SP.
Determined in `_finishWorkout`: `isPrimary = !workoutRepo.hasPrimaryWorkoutToday()`. The first workout of the day is always primary вЂ” regardless of whether it is a daily plan or a custom routine. Custom-only users can build a streak this way. `courseIdIndex` is still `null` for custom routines (not tied to any course).

## Services (Domain Layer)

### SPService
- `forExercise(result, exercise)` вЂ” SP for an exercise; warmup/cooldown (stage=0) в†’ 0
- `forWorkout(...)` вЂ” totals SP; primary = Г—1.5 (daily bonus) Г— 1.1 (completion); bonus = Г—0.5
- `applyToProfile(profile, sp)` вЂ” mutates profile: SP + rank

### StreakService
- `applyWorkout(profile, date)` в†’ bool вЂ” updates streak; on a 1-day gap, uses a freeze. Only called when at least one real exercise (stage > 0, reps > 0 or duration > 0) was performed вЂ” warmup-only workouts do not count.
- `tryAwardFreeze(profile)` в†’ bool вЂ” +1 freeze every 7 days (max 3)
- `isStreakAtRisk(profile)` вЂ” true if no workout today
- `daysSinceLastWorkout(profile)` вЂ” counter

**displayStreak (important):** computed on the fly in `homeDataProvider` without mutating Hive:
```
days <= 1                          в†’ profile.currentStreak
days == 2 && freezeCount > 0       в†’ profile.currentStreak  (freeze will save it on next workout)
otherwise                          в†’ 0
```

### ProgressionService
- `applyResult(progress, exercise, result)` в†’ bool (true = challenge unlocked)
- `advanceStage(progress, nextExercise)` вЂ” stage transition after challenge
- `applyRegression(progress, exercise, daysSkipped)` вЂ” rollback on 3+ days skipped
- `nextExercise(progress)` вЂ” next stage from catalog

### WorkoutGeneratorService
- `generateDailyForCourse({course, courseBranches, progressMap, preferredMinutes, hasPullUpBar})` вЂ” primary method; rotates branches by dayIndex (days since 2020-01-01). N branches: min(2,total) at в‰¤5min, min(3,total) at 10min, total at в‰Ґ15min. Passes `course:` to all `getProgress()` calls.
- `generateDaily(...)` вЂ” legacy wrapper, calls `generateDailyForCourse` with `course: CourseId.calisthenics`.
- `generateChallenge(branch)` вЂ” warmup в†’ current stage (1 easy set) в†’ next stage (challengeTargetReps) в†’ cooldown
- `fromExerciseIds(List<String> ids)` в†’ `WorkoutPlan` вЂ” builds a plan from explicit exercise IDs; uses `startReps/Sets/RestSec`; search order: `byId` в†’ `libraryAll` в†’ `SupplementaryExerciseCatalog.all`
- `hasWarmupAndCooldown(ids)` в†’ `bool` вЂ” checks for `ExerciseTag.warmup` and `ExerciseTag.cooldown` presence
- `addGenericWarmupCooldown(ids)` в†’ prepends `warmup_arm_rotations`, appends `cooldown_shoulder_stretch`

### AchievementService
- `checkAfterWorkout({profile, log, totalWorkouts, achievementRepo})` в†’ `List<String>` new ones
- `checkAfterStageAdvance({branch, newStage, allProgress, achievementRepo})` в†’ `List<String>` new ones

### BleService (singleton)
- `startDiscovery()` вЂ” 30s BLE scan filtered by service UUID `ca11da00-...-0001`; updates `nearbyStream`
- `stopDiscovery()` вЂ” stops scan
- `nearbyStream` вЂ” `Stream<List<NearbyDevice>>` broadcast
- `readProfileJson(NearbyDevice)` вЂ” GATT client read of characteristic `ca11da00-...-0002`; returns decoded `Map<String,dynamic>` or null if remote has no GATT server
- `startAdvertising(profileJson, displayName)` вЂ” initializes `BlePeripheral`, registers GATT service + READ characteristic serving profile JSON bytes, starts BLE advertising with `displayName` as local name
- `stopAdvertising()` вЂ” stops advertising and clears GATT services

**Import note:** `ble_peripheral` exports a class `BleService` вЂ” import the package with prefix `blep` to avoid name collision with our `BleService` singleton.

### FriendsNotifier (StateNotifier)
- `addOrUpdate(FriendProfile)` в†’ `bool` (true = new friend)
- `remove(String id)`
- Provider: `friendsProvider` (`StateNotifierProvider<FriendsNotifier, List<FriendProfile>>`)
- `friendsCountProvider` вЂ” derived `Provider<int>` for Profile screen badge

### QR Profile Exchange
- QR payload: `caliday://friend?data=<base64url_json>`
- JSON fields: `v, id, name, sp, streak, longestStreak, rank, stages, date`
- `FriendProfile.fromQrJson()` parses the decoded JSON

### SoundService (singleton)
- Methods: `tick()`, `ding()`, `pop()`, `complete()`
- **Do NOT use `await player.stop()` before play()** вЂ” causes Android MediaPlayer reset (~100вЂ“200ms). `player.play()` handles restart on its own.
- Configured from `SettingsNotifier` on load and on settings change

### Mutation Pattern
All services **mutate** the passed objects (`UserProfile`, `SkillProgress`) in place.
Persistence is the responsibility of the calling code via repositories.

## Code Style and Conventions

- **Commit messages вЂ” always in English**
- Dart вЂ” official Dart style guide
- Widgets вЂ” prefer `StatelessWidget` + Riverpod `ConsumerWidget`
- File naming: `snake_case`
- Class naming: `PascalCase`
- Public API comments in English
- UI strings via l10n (English primary, Russian secondary; `app_en.arb` is the template)
- No clutter: do not add docstrings/comments to code you are not touching

### Key Patterns

- `activeCourseProvider = StateProvider<CourseId>` initialized from `UserProfile.activeCourse`; switching courses in LibraryScreen updates both the provider and `UserProfile.activeCourseIndex` in Hive
- `homeDataProvider = Provider.autoDispose` + `_ref.invalidate(homeDataProvider)` after workout
- **Always invalidate `displayStreakProvider` before `homeDataProvider`** вЂ” `goroExpressionProvider` keeps it alive via `ref.watch`, so `homeDataProvider`'s `ref.read(displayStreakProvider)` would return a stale cached value otherwise
- **`setHasPullUpBar` must invalidate `homeDataProvider`** вЂ” `activeBranches` is computed from `hasPullUpBar`; without invalidation Pull branch appears only after restart
- `workoutProvider = StateNotifierProvider.autoDispose` вЂ” plan is generated on screen open
- `customWorkoutPlanProvider = StateProvider<WorkoutPlan?>` вЂ” set before navigating to `/workout` for custom routines; takes priority over daily/challenge generation; `courseIdIndex = null`; reset in `_finishWorkout`
- Services mutate HiveObjects in place; persistence is the responsibility of the caller
- `ExerciseResult.completedReps` = rep count from the **last** set (MVP)
- Challenge in a bonus workout: `advanceStage` is called ALWAYS (regardless of `isPrimary`)
- Deep link guard: `if (state.uri.scheme == 'caliday') return '/workout';` in RouterNotifier redirect
- Workout в†’ check `canPop()` before `pop()`: `context.canPop() ? context.pop() : context.go('/home')`
